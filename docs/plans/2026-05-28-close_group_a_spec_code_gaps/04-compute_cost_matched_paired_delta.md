# Task 04 — Compute the cost-matched paired delta

**Plan:** [plan.md](plan.md) · **Status:** Todo

**Implements:** [06-scoring-and-statistics.md](../../benchmark/specs/06-scoring-and-statistics.md) §Confidence intervals and pairwise tests — "Cost-matched delta. The same paired comparison after equalising the budget."
**Depends on:** 03 (build — reuses the per-arm MetricResult helpers and the `cost_matched_resolved_for_arm` machinery)
**Produces:** each of the four pairwise comparisons (A1−A0, A1−A2, A2−A3, A1−A4) produces a cost-matched McNemar delta alongside the raw delta — same arms, but the per-instance bool is `cost ≤ B AND resolved` at the shared equal budget. The cost-matched deltas surface in the rendered ablation report (a new bullet line per pairwise comparison) and as four additional `MetricResult` records emitted by `ablation_metric_results`. Holm-Bonferroni adjustment is applied across the *raw* deltas and the *cost-matched* deltas as two separate families (eight total, two families of four), preserving the spec's α = 0.05 family-wise rate per comparison.
**Pointers:** `benchmark/harness/stats/cost_robustness.py:224` (`equal_budget_for_arms`); `benchmark/harness/stats/cost_robustness.py:246` (`cost_matched_resolved_for_arm`); `benchmark/harness/stats/outcome.py:284` (`mcnemar_delta`); `benchmark/harness/stats/outcome.py:244` (`_instance_resolved_any`, the pairing-reduction rule); `benchmark/harness/stats/ablation_report.py:111` (`PAIRWISE_DELTAS`); `benchmark/harness/stats/ablation_report.py:328` (`holm_bonferroni_adjusted_pvalues`); `benchmark/harness/stats/ablation_report.py:648` (`build_ablation_report`).

## Steps

- [ ] Add an `_arm_instance_cost_matched_resolved(run, arm, budget, basis)` helper to `benchmark/harness/stats/ablation_report.py` that returns a per-instance `dict[str, bool]` whose value is `cost(trial) ≤ budget AND resolved` for any of the arm's trials on that instance — the pairing-reduction is the same `any`-over-trials rule `_instance_resolved_any` uses, so the cost-matched delta is comparable to the raw delta on the same family of instance bools.
- [ ] In `build_ablation_report`, compute the shared budget `B` per pairwise comparison via `equal_budget_for_arms` over **the two arms in the comparison only** (not over all five arms — the spec's "equalising the budget across the two arms" reading). Cache the per-arm cost-matched bool maps so the four comparisons reuse them where the same arm participates.
- [ ] Per comparison, compute a cost-matched `McNemarResult` by passing the cost-matched bool maps through the existing `mcnemar_delta`. The delta value, `b`, `c`, χ² statistic, raw p-value, and exact-vs-asymptotic label all carry through unchanged — the only change is the input bool map.
- [ ] Extend `apply_holm_bonferroni` (or wrap it with a `apply_holm_bonferroni_per_family` helper) so the four raw deltas and the four cost-matched deltas are adjusted as **two separate families** at α = 0.05. Document the choice — the alternative (one family of eight at α = 0.05) is statistically more conservative but conflates the two questions the spec calls for as separate readings (which arm wins on raw resolution vs which arm wins for the same spend).
- [ ] Extend `render_ablation_report` (`benchmark/harness/stats/ablation_report.py:841`) to add a `### Cost-matched pairwise deltas` section after the existing `### Pairwise deltas` section, with one bullet per comparison in the same shape as the existing delta bullets (Δ in pp, χ², raw p, exact/asymptotic label, b/c, Holm-adjusted p, significant-at-α flag). State the cost basis and the equal-budget `B` per comparison in the section preamble.
- [ ] Extend `ablation_metric_results` (from task 03) to emit four additional `MetricResult` records with `metricName = "cost_matched_delta__<label>"` (e.g. `cost_matched_delta__a1_minus_a0`). The four new records carry the delta value (Δ%Resolved) and the Holm-adjusted significance flag in `value`/`ciLow`/`ciHigh` — same shape as the existing delta-row metrics.
- [ ] Tests (`benchmark/tests/test_cost_matched_delta.py`): construct a synthetic five-arm campaign with a known cost distribution where the cheaper arm wins on raw resolution and the dearer arm wins on raw resolution but not for the same spend; assert the cost-matched A1−A0 delta has a different sign or magnitude from the raw delta; assert Holm-Bonferroni adjusts within each family of four (not across all eight); assert the rendered report contains both the existing four pairwise-delta bullets and four new cost-matched ones with the budget `B` stated per bullet.

## Definition of done

- [ ] Four cost-matched McNemar delta rows are computed alongside the existing four raw delta rows; each carries Δ%Resolved, χ², raw p, exact/asymptotic label, b/c, and a Holm-Bonferroni-adjusted p-value at α = 0.05.
- [ ] The two delta families (raw and cost-matched) are adjusted **separately**, so a strong raw signal does not weaken a marginal cost-matched signal or vice versa.
- [ ] The rendered ablation report carries a `### Cost-matched pairwise deltas` section beneath the existing `### Pairwise deltas`, naming the cost basis and the per-comparison equal budget `B`.
- [ ] The new emitter from task 03 returns four additional `MetricResult`s named `cost_matched_delta__<label>` — one per pairwise comparison — and each is schema-valid.
- [ ] Negative-space: an arm pair with no shared scored instances yields a cost-matched delta with `b = c = 0` and a well-defined sign (zero), not a divide-by-zero — the same behaviour `mcnemar_delta` already guarantees for the raw case.
- [ ] Meets the repo definition of done (uv-locked deps, ruff format + lint clean, pyright clean, pytest clean — see `plan.md` baseline). No schema delta required (`MetricResult` already accepts arbitrary `metricName` strings).
- [ ] Reviewable: a reviewer runs the test suite, renders the report on the saved live evidence, and confirms each of the four pairwise comparisons carries both a raw and a cost-matched delta bullet, with the budget `B` printed alongside.

## Open questions

- *Family for Holm-Bonferroni.* This task pins two families of four at α = 0.05 each (raw deltas; cost-matched deltas). The alternative — one family of eight — is statistically more conservative and would weaken every signal by a factor of two. The choice should be documented as a `Decision` on `06-scoring-and-statistics.md` if the user prefers the one-family reading; the code constant `HOLM_BONFERRONI_ALPHA` already lives there and can be left unchanged either way.
